## 2024-04-13 - [Preventing CSV Injection in Exports]
**Vulnerability:** A CSV injection vulnerability existed in `src/pages/TutorDashboard.tsx` where user-provided input (task titles, descriptions, etc.) was exported directly to a CSV file. If this input started with characters like `=`, `+`, `-`, or `@`, spreadsheet applications like Excel could interpret the data as formulas instead of text, potentially leading to arbitrary command execution or data exfiltration on the end-user's machine.
**Learning:** Even though the web application safely handles text, data exported to other formats (like CSV) can become malicious in the context of the application that opens it. Sanitization must consider the context where the data is consumed, not just where it is generated.
**Prevention:** When building custom CSV export functions, always sanitize field values. For fields that start with dangerous characters (`=`, `+`, `-`, `@`, `\t`, `\r`), prepend a single apostrophe (`'`). This tells spreadsheet applications to treat the cell contents as plain text rather than an executable formula.
